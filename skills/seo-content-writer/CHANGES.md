# CHANGES.md — SEO Content Writer Distillation

## What changed

### Structure: proposal → operational skill
- **Before:** Written as "proposed changes to add" — not a standalone skill
- **After:** Self-contained framework, ready to execute without external context

### Rigor principles applied

| Principle | Implementation |
|-----------|---------------|
| **plan-gate** | Added mandatory pre-flight table: keyword, intent, audience, competitors. No confirmation → no writing. |
| **adversarial-verify** | Post-draft checklist with measurable targets (density 1–2%, zero banned patterns, link counts) |
| **live-state-truth** | Requires `web_search` competitor scan before writing; no assumed keyword data |
| **scope-fence** | Explicit rule: single keyword only, no bonus topics, every paragraph must serve the intent |
| **ruthless-editor** | Replaced prose sections with tables; removed code fences wrapping content; cut redundancy |
| **memory-hygiene** | Added structured log format: title, keyword, density, word count, link counts |

### Content preserved
- All 12 banned patterns kept (reformatted as table)
- SEO rules: title ≤60, meta ≤155, H1 unique, keyword in first 100 words
- Quality requirements: ≤4 sentence paragraphs, data with sources, no repetition
- Example-before-explanation rule

### Content removed
- "Related Skills" cross-references (not operational, just suggestions)
- Nested markdown code fences (skill was instructions-about-instructions)
- "Mudanças Propostas" framing (no longer a proposal)
- Conversion-specific checklist items merged into adversarial-verify
