# 2026-04-20 - Fix Lentidão OpenClaw Startup

## 🐛 Problema
O OpenClaw estava demorando muito para inicializar por causa do Bonjour/mDNS ficando preso no estado "announcing". O watchdog detectava e reiniciava repetidamente, causando delays de 77+ segundos.

### Logs do problema:
```
[bonjour] watchdog detected non-announced service; attempting re-advertise (state=probing)
[bonjour] restarting advertiser (service stuck in announcing for 77790ms)
```

## ✅ Solução
Adicionado `OPENCLAW_DISABLE_BONJOUR=1` no `.env` do OpenClaw.

**Por quê funciona:** O Bonjour (mDNS) é usado para discovery local na rede. Como usamos canais diretos (Telegram, etc), não precisamos dele. No Windows, a biblioteca `@homebridge/ciao` tem problemas com interfaces de rede que não suportam multicast.

### Resultado:
- **Antes:** hooks 03:41:36 → bonjour loop até 03:43:56 (~2min30 de delay)
- **Depois:** hooks 03:49:20 → **ZERO mensagens bonjour**, startup limpo em ~34s
